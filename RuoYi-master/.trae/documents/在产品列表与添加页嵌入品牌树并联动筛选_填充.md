## 目标
- 将“品牌→分类→商品”树嵌入产品列表页与添加页左侧，效果对齐用户管理的组织结构树。
- 列表页：点击树节点联动右侧表格筛选（按品牌/分类）。
- 添加页：点击树节点自动填充品牌与分类选择框。

## 技术方案
- 统一树数据接口：`GET /product/tree/zTree`
  - 返回 zTree 需要的节点结构：`[{id,name,pId,type,realId,brandId}]`
  - 节点 ID 采用分段避免冲突：品牌节点 `id=100000+brandId`，分类节点：若有父分类 `pId=200000+parentId`，顶级分类 `pId=100000+brandId`；分类节点自身 `id=200000+categoryId`
  - 节点属性：`type='brand'|'category'`；分类节点含 `brandId` 与真实 ID `realId`
- 左侧布局：复用 `ui-layout-west` + `ztree`（引入 `layout-latest-css/js` 与 `ztree-css/js`），对齐 `system/user/user.html` 的结构与交互。

## 页面改造
- 产品列表页 `product/info/info.html`
  - 左侧加入 zTree 容器与工具栏（展开/折叠/刷新）
  - 表单中增加隐藏字段 `brandId` 与 `categoryId`
  - 初始化树：`$.tree.init({url: ctx+'product/tree/zTree', onClick(zOnClick)})`
  - 点击逻辑：
    - 若品牌节点：设置 `brandId`（清空 `categoryId`），触发 `$.table.search()`
    - 若分类节点：设置 `brandId` 与 `categoryId`，触发 `$.table.search()`
- 产品新增页 `product/info/add.html`
  - 左侧加入 zTree 容器（同样初始化）
  - 点击逻辑：
    - 若品牌节点：设置品牌下拉框到对应品牌（按 id 或名称匹配）
    - 若分类节点：设置品牌与分类下拉框到对应项
